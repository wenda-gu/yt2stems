# License notes

This project currently uses the MIT license.

## Reasonable starting points

- MIT: simple and permissive
- Apache-2.0: permissive with explicit patent language
- GPL-3.0-only: stronger copyleft if you want derivative works to stay open under the same license family

## Practical guidance

- Choose MIT if you want the lowest-friction adoption
- Choose Apache-2.0 if you want a permissive license with stronger legal clarity
- Choose GPL only if strong copyleft is an intentional project goal

If you ever want to change the license later, update `LICENSE`, `pyproject.toml`, and the README at the same time.
